Persistent Sets¶

A persistent set is created by calling burrahobbit.set() which returns an object of the PersistentTreeSet type (see below for a documentation of its methods). burrahobbit.set() behaves analogous to the builtin set() function with regards to the parameters it takes and how it interprets them.

In addition to the methods described below, sets implement the binary operators &, | and ^ (and, or, xor): a & b returns a new value of b‘s type consisting of all items of b whose key is also present in a; a | b returns a updated with the elements in b; a ^ b returns a new persistent set with all items that are only contained in exactly one of them. The return value of a & b is always the same type as b is (the rationale for this behaviour is that a & b only contains nodes from b).

class burrahobbit.treeset.PersistentTreeSet(root=<burrahobbit._tree.NullNode object at 0x2aad04dde5a0>)¶
add(key)¶

Return copy of self with an association between key and value. May override an existing association.

transient()¶

Return transient (mutable) copy of self. Changing the copy will not affect the original object’s immutability.

See TransientTreeSet.

without(key)¶

Return copy of self with key removed.

class burrahobbit.treeset.TransientTreeSet(root=<burrahobbit._tree.NullNode object at 0x2aad04dde5a0>)¶

All methods of PersistentTreeSet available, with the addition of persistent().

persistent()¶

Return a persistent version of self.

CAUTION: The TransientTreeMap MAY NOT BE USED after calling this method.
