Persistent Dicts¶

A persistent dict is created by calling burrahobbit.dict() which returns an object of the PersistentTreeMap type (see below for a documentation of its methods). burrahobbit.dict() behaves exactly the way the builtin dict function of Python does, barring that it returns instances of PersistentTreeMap instead of regular dicts.

In addition to the methods described below, dictionaries implement the binary operators &, | and ^ (and, or, xor): a & b returns a new persistent dictionary containing a mapping from all the keys that are contained in both a and b to the respective values in b; a | b returns a updated with the mappings in b (if a key is defined in both a and b, it is mapped to the value in b); a ^ b returns a new persistent dictionary with all items whose key is only contained in exactly one of them. The return value of a & b is always the same type as b is (the rationale for this behaviour is that a & b only contains nodes from b).

API Reference¶

class burrahobbit.treedict.PersistentTreeMap(root=<burrahobbit._tree.NullNode object at 0x2aad04dde5a0>)¶
assoc(key, value)¶

Return copy of self with an association between key and value. May override an existing association.

iteritems()¶

Yield key, value pairs for all items.

iterkeys()¶

Yield keys for all items.

itervalues()¶

Yield values for all items.

transient()¶

Return transient (mutable) copy of self. Changing the copy will not affect the original object’s immutability.

See TransientTreeMap.

without(key)¶

Return copy of self with key removed.

class burrahobbit.treedict.TransientTreeMap(root=<burrahobbit._tree.NullNode object at 0x2aad04dde5a0>)¶

TransientTreeMaps are used if - and only if - one function does so many changes to a PersistentTreeMap the immutability would prove inefficient.

The function has to return transienttreemap.persistent() in order to ensure that the treemap cannot be changed afterwards.

All methods of PersistentTreeMap available, with the addition of persistent().

persistent()¶

Return a persistent version of self.

CAUTION: The TransientTreeMap MAY NOT BE USED after calling this method.
